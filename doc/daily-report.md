# Daily Report 技能文档

## 使用时机

在以下情况使用此技能：

- 用户请求生成日报、工作总结、提交记录汇总
- 手动调用 `/daily-report` 命令
- 用户询问"今天做了什么"、"工作进度"等
- 需要快速汇总当日 Git 提交记录

---

## 核心功能

- 获取 Git 用户信息（用户名、邮箱）
- 支持今天、昨天、自定义日期统计
- 按 project scope 分组提交记录
- 可选统计信息（提交次数、新增/删除行数）
- 输出标准化日报格式

---

## 执行步骤

### 步骤 1：获取 Git 用户信息

```bash
git config user.name
git config user.email
```

用于筛选当前用户的提交记录，避免包含他人的提交。

---

### 步骤 2：询问统计日期

使用 `AskUserQuestion` 让用户选择统计日期：

| 选项 | 说明 |
|------|------|
| 今天（默认） | 当天日期，格式 YYYY-MM-DD |
| 昨天 | 昨天日期，格式 YYYY-MM-DD |
| 自定义日期 | 用户输入指定日期 |

---

### 步骤 3：询问统计信息

使用 `AskUserQuestion` 让用户选择是否包含统计信息：

| 选项 | 说明 |
|------|------|
| 否（默认） | 仅输出提交内容 |
| 是 | 包含提交次数、新增/删除行数 |

---

### 步骤 4：获取提交记录

#### 基础提交记录

```bash
git log --author="<用户名>" --since="<日期> 00:00:00" --until="<日期> 23:59:59" --format="%h %s"
```

#### 包含统计信息时

```bash
git log --author="<用户名>" --since="<日期> 00:00:00" --until="<日期> 23:59:59" --shortstat --format="%h %s"
```

---

### 步骤 5：解析并分组

按 project scope 分组提交记录：

#### Commit 解析规则

- 格式：`<type>(<scope>): <subject>` 或 `<type>: <subject>`
- 提取 scope 作为项目分组依据
- 提取 subject 作为工作内容描述
- 无 scope 的归入"其他"分组
- 同一 scope 的多个 subject 用分号连接

#### 解析示例

| Commit 消息 | Scope | Subject |
|-------------|-------|---------|
| `feat(hr-delivery): 新增考评卡片` | `hr-delivery` | `新增考评卡片` |
| `fix(perf): 修复绩效计算逻辑` | `perf` | `修复绩效计算逻辑` |
| `docs: 更新项目文档` | `其他` | `更新项目文档` |

---

### 步骤 6：生成日报

#### 基础格式（不含统计）

```text
日报 (日期)

scope名称：提交内容1；提交内容2
其他：无scope的提交内容

统计：共 X 次提交
```

#### 包含统计格式

```text
日报 (日期)

scope名称：提交内容1；提交内容2
其他：无scope的提交内容

统计：共 X 次提交，新增 Y 行，删除 Z 行
```

---

## 统计信息解析

### 行数统计

使用 `--shortstat` 输出格式：

```text
2 files changed, 50 insertions(+), 10 deletions(-)
```

解析规则：
- `insertions(+)` → 新增行数
- `deletions(-)` → 删除行数

## 输出示例

### 基础日报

```text
日报 (2026-04-29)

hr-delivery：新增考评卡片；修复绩效计算逻辑
git-commit：完善技能支持子仓库和拉取流程
daily-report：添加工作日报生成技能
其他：更新项目文档

统计：共 5 次提交
```

### 包含统计信息

```text
日报 (2026-04-29)

hr-delivery：新增考评卡片；修复绩效计算逻辑
git-commit：完善技能支持子仓库和拉取流程
daily-report：添加工作日报生成技能
其他：更新项目文档

统计：共 5 次提交，新增 120 行，删除 35 行
```

## 技术要点

| 要点 | 说明 |
|------|------|
| 用户名获取 | 使用 `git config user.name` |
| 行数统计 | 使用 `git log --shortstat` |
| 日期格式 | 统一为 YYYY-MM-DD |
| 时间范围 | 00:00:00 至 23:59:59 |
| 分组依据 | Commit message 的 scope 部分 |

---

## 扩展功能（可选）

### 跨仓库日报

可扩展支持跨多个 Git 仓库的日报生成：

1. 用户指定多个仓库路径
2. 依次获取每个仓库的提交记录
3. 合并分组输出汇总日报

### 输出格式

```text
日报 (2026-04-29)

[仓库A] hr-delivery：新增考评卡片
[仓库B] api：优化接口响应速度
其他：更新项目文档
```
