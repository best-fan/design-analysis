# 分析基础

## 一、分析顺序（必守）

**必须按以下顺序阅读与记录设计稿，否则容易产生疏漏。**

### 1. 从上到下

先按垂直方向，从页面最上方扫到最下方，确定所有横向「带」与区块的先后顺序。

**为什么重要**：
- 与前端布局顺序一致，避免实现时顺序颠倒
- 确保顶部或首屏的大字号文案不被遗漏

### 2. 从左到右

在同一行或同一层级内，再按从左到右依次记录，不跳过、不凭印象合并。

**为什么重要**：
- 避免只记录左侧内容而漏掉右侧或对称区域
- 确保同一层级内的元素顺序正确

### 3. 从外到里

在每个区块内部，按「外层容器 → 内层容器 → 叶子元素」逐层进入，先轮廓后细节。

**为什么重要**：
- 确保层级嵌套关系正确，不丢层或顺序颠倒
- 先理解整体结构，再深入细节

## 二、四类重中之重

**文字、图片、布局、层级是页面元素的四类重中之重，均不允许出错或遗漏。**

### 文字

**所有可见文案都必须记录**，包括：标题、副标题、说明文字、按钮/链接文案、占位文案、Tab 文字、标签文字。

每处文字需记录：内容、字体、字号、字重、颜色、行高。

**易漏点**：页面最上方的欢迎语/大标题、副标题、按钮/链接文案、占位提示文字、Tab 或标签文字。

### 图片

**所有图片、占位图、图标区域都必须记录**。

每处图片需记录：尺寸、位置、比例、圆角、与相邻元素的间距、多图拼接时的布局关系。

**易漏点**：小图标、占位图、多图拼接时的布局关系、图片的圆角和比例。

### 布局

**区域与元素的位置、尺寸、间距、对齐方式、排列方向必须准确记录。**

记录项：位置（x, y）、尺寸（width, height）、间距（padding, gap, margin）、对齐方式、排列方向、多图拼接方式。

**易漏点**：间距的方向性、对齐方式、排列方向、多图拼接时的布局方式。

### 层级

**从外到里的嵌套关系、区块的先后顺序与叠放关系必须写清。**

**⚠️ 强制要求：必须使用 MCP 工具读取实际节点树，禁止凭视觉推断层级结构！**

记录项：嵌套关系、先后顺序、叠放关系、每个子节点的坐标和尺寸。

## 三、常见遗漏检查点

分析时必须逐一检查：

### 四类重中之重（必查）

- [ ] **所有可见文字**：欢迎语、大标题、副标题、说明、按钮/链接/占位/Tab 文案等
- [ ] **所有图片/占位图**：尺寸、位置、比例、多图拼接布局
- [ ] **布局**：位置、尺寸、间距、对齐、排列方向与设计稿一致
- [ ] **层级**：从外到里的嵌套与先后顺序正确，无丢层或顺序颠倒

### 区域遗漏强制检查（必须执行）

**这是最常见的遗漏类型**：

- [ ] **统计设计稿顶层节点数量**：使用设计稿工具获取所有顶层节点
- [ ] **按 y 坐标排序所有区域**：从 y=0 开始，逐一向下扫描
- [ ] **确认底部区域**：特别检查 y > 首屏高度（如 1080px）的区域
- [ ] **区域数量对比**：文档记录数 = 设计稿实际区域数

### 样式相关

- [ ] 背景色、边框、阴影
- [ ] 内边距与外边距（padding / gap / margin）
- [ ] 对齐方式（左对齐 / 居中 / 右对齐）

### 表格区域专项

- [ ] **表格列标题**：逐列读取实际文字，**禁止推断**
- [ ] **表格列数**：确认列数准确，包括复选框列
- [ ] **特殊列标注**：含头像、进度条、标签、状态指示器的列
