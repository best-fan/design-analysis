# 异常修正强制流程

## 触发条件

以下情况触发异常修正流程：

| 触发条件 | 示例 |
|----------|------|
| 元素坐标不在归属区域内 | 年份选择器 x=546 被误判为属于标题 x=714 区域 |
| 区域合并错误 | 独立组件被合并到相邻区域 |
| 边界验证表与区域详情不一致 | 验证表改了，详情没改 |

---

## 强制执行步骤

**发现异常后，依次执行以下 5 步：**

```
步骤1: 记录异常发现位置
       → 明确记录：哪个元素、坐标多少、错误归属哪里

步骤2: 更新区域边界验证表
       → 修正边界值、更新元素归属验证

步骤3: 🔴🔴 同步更新区域详细分析
       → 必须同步修改以下内容：
         - 区域概述中的位置描述
         - 布局参数中的坐标说明
         - 层级结构中的归属关系
         - 如需拆分区域：顺延后续编号

步骤4: 更新整体布局结构图
       → ASCII 布局图反映修正后的位置关系

步骤5: 执行修改完整性校验
       → 使用 Grep 搜索异常元素，确认所有提及处均已修正
```

---

## 修改完整性校验清单

**修正完成后，执行以下校验：**

| 校验项 | 执行方法 | 通过标准 |
|--------|----------|----------|
| 边界表一致性 | 对比验证表与区域详情的坐标 | 坐标一致 |
| 编号连续性 | Grep 搜索 `### 3.` 检查编号 | 无跳号/重复 |
| 布局图一致性 | 布局图元素位置与实际坐标匹配 | 位置正确 |
| 层级归属一致性 | 层级结构与边界验证表一致 | 归属一致 |

---

## 违规后果

| 违规行为 | 后果 |
|----------|------|
| 只改验证表不改区域详情 | 校验二十失败，文档无效 |
| 拆分区域后未顺延编号 | 编号混乱，校验十失败 |
| 未执行修改完整性校验 | 可能遗漏其他需要修正处 |
