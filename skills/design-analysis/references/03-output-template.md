# 输出模板

## 文档路径

```
openspec/ui-checklist/{序号}-{模块名}.md
```

### 序号规则

- 三位数字递增（001, 002, 003...）
- 新序号 = 现有最大序号 + 1
- 空目录从 001 开始

### 模块名规则

- 英文 kebab-case
- 示例：`login-page`、`dashboard-header`

---

## ⚠️ 不输出到文档的内容

以下内容是**分析过程中的工具**，用于确保分析准确性，**不需要在最终输出文档中显示**：

| 内容 | 说明 |
|------|------|
| 区域边界验证表 | 分析阶段用于验证元素归属，发现问题及时修正，不输出到文档 |
| 校验详细记录表格 | 校验过程记录表格，只在校验记录中简要说明已通过项 |
| 禁用列（`enabled: false`） | 表格列分析中不记录禁用列，只记录 `enabled: true` 的可见列 |

**校验记录只需简要列出已通过的校验项，无需详细展开表格。**

## 强制要求

- **禁止遗漏任何章节**：必须包含一~九所有章节
- **禁止使用自创格式**：章节标题、内容结构必须与模板完全一致
- **禁止跳过校验步骤**：生成后必须执行校验
- **禁止记录禁用列**：`enabled: false` 的表格列不在文档中记录

- **图表区域必须包含 `#### 图表：<图表名称>` 小节**
