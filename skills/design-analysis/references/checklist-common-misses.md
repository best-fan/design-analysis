# 常见遗漏检查点

## 概述

本文档列出设计稿分析时常见的遗漏点，分析时必须逐一检查，确保分析完整。

## 检查清单

分析时必过一遍：

### 四类重中之重（必查）

- [ ] **所有可见文字**：欢迎语、大标题、副标题、说明、按钮/链接/占位/Tab 文案等，按从上到下逐条核对
- [ ] **所有图片/占位图**：尺寸、位置、比例、多图拼接布局，无遗漏
- [ ] **布局**：位置、尺寸、间距、对齐、排列方向、多图拼接方式与设计稿一致，无错位或比例失调
- [ ] **层级**：从外到里的嵌套与先后顺序正确，无丢层或顺序颠倒

详见 `analysis-priorities.md` 获取详细说明。

### 样式相关

- [ ] 背景色（页面、卡片、区块、按钮）
- [ ] 边框（颜色、宽度、圆角）
- [ ] 阴影（颜色、模糊、偏移）
- [ ] 内边距与外边距（padding / gap / margin）
- [ ] 对齐方式（左对齐 / 居中 / 右对齐）

### 区域相关

- [ ] **右侧或对称区域**（易只写左侧漏右侧）
- [ ] **底部或折叠区域**
- [ ] **区域总数校验**：文档记录的区域数量与设计稿顶层节点数量一致

### ⚠️ 区域遗漏强制检查（必须执行）

**这是最常见的遗漏类型，必须严格执行以下检查**：

- [ ] **统计设计稿顶层节点数量**：使用设计稿工具获取所有顶层 GROUP/FRAME/INSTANCE 节点
- [ ] **按 y 坐标排序所有区域**：从 y=0 开始，逐一向下扫描
- [ ] **确认底部区域**：特别检查 y > 首屏高度（如 1080px）的区域
- [ ] **区域数量对比**：文档记录数 = 设计稿实际区域数

**检查方法**：
```
1. 获取设计稿数据（如 MasterGo DSL）
2. 遍历根节点的所有直接子节点
3. 过滤出 GROUP/FRAME/INSTANCE 类型且有实际尺寸的节点
4. 统计总数
5. 与文档中「区域列表」的行数对比
6. 若不一致，立即补充遗漏区域
```

**常见遗漏场景**：
| 场景 | 问题原因 | 避免方法 |
|------|----------|----------|
| 只分析首屏 | 忽略 y > 1080 的区域 | 强制遍历所有顶层节点 |
| 依赖 Agent 结果 | Agent 可能遗漏底部区域 | 亲自验证节点数量 |
| 推测区域数量 | 根据「常见布局」假设 | 必须从设计稿提取实际数据 |

### 元素相关

- [ ] 小图标、小按钮、Tab 文字
- [ ] 占位图/占位图标尺寸与比例

### 表格区域专项（必查）

- [ ] **表格列标题**：逐列读取实际文字，**禁止推断**（如看到表格就假设有"项目编号、项目名称"等常见列）
- [ ] **表格列数**：确认列数准确，包括复选框列（可能无标题文字）
- [ ] **表格列宽**：记录每列宽度，带单位的精确值
- [ ] **特殊列标注**：含头像、进度条、标签、状态指示器等复合元素的列需特别标注
- [ ] **表头层级**：确认是否有分组表头（多层级表头）

**警示**：表格列标题是高频错误区，必须**逐列读取设计稿中的实际文字**，不可根据页面名称或经验推断。

## 详细说明

### 文字遗漏

**易漏的文字类型**：
- 页面最上方的欢迎语/大标题（最容易在「只记了下面小字」时被漏掉）
- 副标题
- 按钮/链接文案
- 占位提示文字
- Tab 或标签文字

**检查方法**：按「从上到下」扫到时立即记下，不要等到后面再补。

### 图片遗漏

**易漏的图片类型**：
- 小图标
- 占位图
- 多图拼接时的布局关系
- 图片的圆角和比例

**检查方法**：逐区域扫描，确保每张图片都有记录。

### 布局遗漏

**易漏的布局参数**：
- 间距（padding/gap/margin）的方向性
- 对齐方式（左对齐/居中/右对齐）
- 排列方向（横向/纵向/网格）
- 多图拼接时的布局方式

**检查方法**：对照设计稿，确保布局参数与设计稿一致。

### 层级遗漏

**易漏的层级问题**：
- 外层容器被忽略
- 嵌套层级丢失
- 顺序颠倒
- 叠放关系不明确

**检查方法**：按「从外到里」逐层分析，确保每层都有记录。

### 区域遗漏

**易漏的区域**：
- **右侧或对称区域**：只记录了左侧，漏掉右侧
- **底部或折叠区域**：只关注首屏，漏掉底部内容

**检查方法**：按「从上到下、从左到右」完整扫描，确保所有区域都有记录。

## 使用建议

1. **分析时使用**：在第二步「区域与元素提取」时，对照此检查清单逐项检查
2. **完成后验证**：分析完成后，再次对照此检查清单验证是否完整
3. **文档中记录**：在 UI 分析清单的「六、常见遗漏检查点」章节中，列出本次分析已核对的项
