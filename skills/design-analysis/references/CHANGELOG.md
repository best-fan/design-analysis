## [1.23.0] - 2026-04-09

### Changed

- **统一禁用列输出规则**：明确禁用列（`enabled: false`）不在文档中记录
- 更新 `02-workflow-main.md` 状态字段检查范围和强制规则，移除矛盾的"标注禁用"要求
- 更新 `04-verification.md` 校验六强制规则，统一为"不在文档中记录"
- 更新 `03-output-template.md` "不输出到文档的内容"章节，添加禁用列说明
- 更新 `SKILL.md` 禁止行为列表，新增禁用列相关规则
- 更新 `SKILL.md` "输出文档不包含的内容"章节，添加禁用列说明

### Fixed

- 修复禁用列规则矛盾问题：之前部分规则要求"标注禁用"，部分规则要求"不记录"
- 删除矛盾的规则说明，统一为：禁用列不在文档中记录

### Purpose

**统一禁用列处理规则，避免前端开发实现不需要的功能**

| 问题 | 之前的矛盾规则 | 统一后的规则 |
|------|---------------|-------------|
| 表格禁用列处理 | 部分要求"标注禁用"，部分要求"不记录" | 统一：不在文档中记录 |
| 文档输出 | 列表中包含禁用列 | 只记录 enabled: true 的列 |
| 区域边界验证表 | 部分文档中包含此表 | 明确不输出到文档 |

**改进核心**：
> 禁用列（`enabled: false`）在设计稿中不可见，前端开发不会实现，因此不应在 UI 分析清单文档中记录。
>
> 这样可以避免开发误解，实现不存在的功能。

### Changed

- 更新 SKILL.md 版本号至 1.22.0
- 强化表格列分析的强制执行机制，要求系统性遍历所有列节点

### Purpose

**防止表格列遗漏问题再次发生**

| 问题 | 根本原因 | 改进措施 |
|------|----------|----------|
| 「项目名称」列未记录 | 只凭部分数据推断，未系统遍历所有列 | 新增强制步骤：表格列系统性遍历 |
| 表头内容是占位符「Title」被跳过 | 凭经验判断，认为占位符不重要 | 新增强制规则：表头内容是占位符时仍需记录该列存在 |

**改进核心**：
> 表格是最高频错误区域，必须系统性遍历所有列节点，不能凭经验跳过任何列。
>
> 本次改进的核心是：**强制要求逐列遍历表格列节点，并验证列宽总和≈表格宽度**。

### Mandatory Rules

- [ ] **必须获取表格 columns 节点的完整 children 列表**
- [ ] **必须对每个列节点逐一读取，禁止凭经验跳过**
- [ ] **必须读取每个列节点的表头单元格内文字，不能凭列名推断**
- [ ] **表头内容是占位符（如「Title」）时，仍需记录该列存在**
- [ ] **必须检查每个列节点的 `enabled` 字段，不是表头单元格的字段**
- [ ] **必须验证列宽总和 ≈ 表格宽度**

### Real Case

**案例：MasterGo INSTANCE 组件内部元素遗漏（2026-04-08）**

| 问题 | 根本原因 | 解决方案 |
|------|----------|----------|
| 年份选择器遗漏 | 未递归遍历 INSTANCE 内部 | 新增 `extract_instance_texts()` 函数 |
| Tab 切换文字为空 | 只列出结构，未提取文字 | 强制要求提取 INSTANCE 内部所有 TEXT 节点 |
| 层级结构空泛 | 未验证实际内容 | 禁止出现「Tab组件，文字待业务填充」等模糊描述 |

**根本原因分析**：
```
1. MasterGo 的实际内容存储在 INSTANCE 组件内部，不在顶层 GROUP 节点
2. INSTANCE 节点的 children 包含实际的标题、选择器、Tab 文字等 TEXT 节点
3. 常规深度遍历默认跳过 INSTANCE 内部结构
4. 脚本 find_common_components() 只检查直接 children，未递归进入 INSTANCE 内部
```

**改进措施**：
1. 在 `05-tools-guide.md` 新增专门的「MasterGo INSTANCE 组件深度遍历规范」章节
2. 在 `04-verification.md` 新增「校验二十三：INSTANCE 内部元素完整性校验」
3. 更新脚本添加 `extract_instance_texts()` 函数，强制递归遍历 INSTANCE 内部
4. 新增三条禁止行为，用 ⚠️⚠️⚠️ 标记强调

**强制规则**：
- [ ] 每个区域必须检查是否包含 INSTANCE 节点
- [ ] INSTANCE 节点必须递归遍历其内部 children（深度至少 4 层）
- [ ] 所有文字必须从 TEXT 节点的 text 字段提取
- [ ] 年份选择器必须搜索「年」「月」「日」关键词
- [ ] Tab 切换文字必须从 INSTANCE 内部提取，不能留空
- [ ] 层级结构中禁止出现空的节点结构

### Real Case

**案例：xmjl.pen 年份选择器归属错误（2026-04-08）**

| 问题 | 设计稿坐标 | 错误归属 | 正确归属 | 根本原因 |
|------|------------|----------|----------|----------|
| 年份选择器 | x=546 | 右侧图表区(x≥698) | 左侧卡片区(160≤x<682) | 凭视觉直觉判断，未验证坐标边界 |

**验证过程**：
- 左侧卡片区边界：x=160, 右边界=160+522=682
- 验证：160 ≤ 546 < 682 ✅ → 属于左侧卡片区
- 右侧图表区边界：x=698
- 验证：546 < 698 ❌ → 不属于右侧图表区

**教训**：
1. **禁止凭视觉直觉判断元素归属**
2. **必须验证元素坐标是否在区域边界内**
3. **左右布局中，边界附近的元素必须重点验证**
4. **层级结构中，必须标注独立节点的坐标位置**
