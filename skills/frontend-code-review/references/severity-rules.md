# 问题严重等级规则

## 等级定义

### 🔴 严重（Critical）

**定义**：必须立即修复，阻止代码合并。

**判定标准**：

1. **安全漏洞**
   - 已知的高危安全漏洞
   - XSS、SQL 注入等安全风险
   - 敏感信息泄露

2. **运行时错误风险**
   - 使用 `any` 类型导致类型检查失效
   - 空指针引用风险
   - 未处理的 Promise rejection

3. **规范严重违反**
   - 禁止的代码模式（如直接修改 props）
   - 核心架构违反（如类型定义散落在业务文件中）

**处理要求**：
- 必须在合并前修复
- 无法绕过或忽略

---

### 🟡 中等（Medium）

**定义**：建议在本次迭代中修复，不影响紧急发布。

**判定标准**：

1. **代码质量问题**
   - 函数超过 50 行
   - 嵌套超过 3 层
   - 代码重复

2. **类型安全问题**
   - 类型定义不规范（如 interface 未使用 `I` 前缀）
   - 从 `@/types/` 子路径直接导入

3. **Vue 规范问题**
   - 未使用 `defineOptions` 声明组件名称
   - Props 默认值未使用工厂函数

4. **依赖问题**
   - 中危安全漏洞
   - 依赖版本过时（主版本落后）

**处理要求**：
- 建议在当前迭代修复
- 特殊情况可后续修复（需说明原因）

---

### 🟢 轻微（Minor）

**定义**：可后续优化，不阻塞发布。

**判定标准**：

1. **命名不规范**
   - 变量命名不够语义化
   - 文件命名不符合约定

2. **代码风格**
   - 注释不完整
   - 格式不一致

3. **低危问题**
   - 低危安全漏洞
   - 依赖版本过时（次版本/补丁版本落后）

4. **性能优化**
   - 可优化但影响小的性能问题
   - 未使用 computed 但计算简单

**处理要求**：
- 记录到技术债务清单
- 后续迭代逐步优化

---

## 报告中的问题排序

1. 按严重等级排序（严重 > 中等 > 轻微）
2. 同等级内按类别排序（安全 > 类型 > 代码质量 > Vue > 样式）
3. 同类别内按文件路径排序

## 特殊情况处理

### 可接受的例外

以下情况可标记为「已知例外」，不计入问题：

1. **第三方库限制**
   - 第三方库要求使用 `any`
   - 需添加注释说明原因

2. **历史代码**
   - 已有代码存在历史问题
   - 仅对新代码进行审查

3. **临时方案**
   - 临时解决方案需要在注释中说明
   - 添加 `TODO` 或 `FIXME` 标记

### 例外标记方式

```typescript
// eslint-disable-next-line @typescript-eslint/no-explicit-any
// @review-exemption: 第三方库 xxx 缺乏类型定义
const data: any = thirdPartyLib.getData()
```

### 审查报告中的例外说明

```markdown
#### 例外说明

| 文件 | 问题描述 | 例外原因 |
|------|---------|---------|
| src/utils/helper.ts:42 | 使用 any 类型 | 第三方库缺乏类型定义 |
```
