# 更新日志

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)。

## [1.2.0] - 2026-03-26

### 新增

- 添加「项目配置」章节，说明默认配置和外部规则文件处理
- 添加错误处理流程，覆盖静态检查和 npm 安全检查的异常情况
- 添加 `references/CHANGELOG.md` 版本更新记录

### 优化

- 精简 SKILL.md 内容，移除与 references 重复的部分
- 严重等级详情统一在 `references/severity-rules.md` 中维护
- 审查要点详情统一在 `references/review-checklist.md` 中维护

## [1.1.0] - 2026-03-26

### 新增

- 新增初始化步骤（步骤 1），包含外部规则文件检查和报告目录检查
- 在每个步骤中明确「读取规范」时机
- 添加外部规则文件处理逻辑（检查存在、读取应用、跳过标注）

### 修复

- 移除命令参数格式章节，改为通过交互选择审查范围
- npm 安全检查强制使用官方 registry（避免私有 registry 不支持 audit 接口）

---

## [1.0.0] - 2026-03-24

### 新增

- 初始版本
- 支持 5 类审查：代码质量、类型安全、Vue 最佳实践、样式规范、npm 安全
- 输出结构化审查报告
- 支持全量审查、npm 安全检查、指定目录/文件、变更文件 4 种审查范围
