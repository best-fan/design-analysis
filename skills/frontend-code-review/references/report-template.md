## 格式校验清单

> ⚠️ 生成报告后，必须对照以下清单检查完整性。

### 必须包含的章节

| 章节 | 必要性 | 检查项 |
|------|--------|--------|
| 报告头部 | ✅ 必须 | 时间、审查人、范围、文件数、类型 |
| 摘要统计 | ✅ 必须 | 统计表格完整，总计行正确 |
| 问题列表 | ✅ 必须 | 按严重等级分组，编号连续 |
| npm 安全检查 | ⚡ 条件必须 | 执行安全检查时必须包含 |
| 静态检查输出 | ✅ 必须 | ESLint 和 TypeScript 原始输出 |
| 审查文件列表 | ✅ 必须 | 所有审查文件已列出 |
| 总结与建议 | ✅ 必须 | 优先处理建议完整 |
| 例外说明 | ⚡ 可选 | 有例外时才需要 |

### 每个章节的必要元素

#### 报告头部
- [ ] 审查时间格式：`YYYY-MM-DD HH:mm:ss`
- [ ] 审查范围：文件路径或目录
- [ ] 审查文件数：数字
- [ ] 审查类型：全量审查 / 暂存审查 / 安全检查

#### 摘要统计
- [ ] 表格包含：类别、严重、中等、轻微、总计
- [ ] 类别行：代码质量、类型安全、Vue 规范、样式规范、npm 安全
- [ ] 总计行：各类别数字之和
- [ ] 静态检查结果：ESLint 和 TypeScript 状态

#### 问题列表
- [ ] 按严重等级分组：🔴 严重 → 🟡 中等 → 🟢 轻微
- [ ] 每个问题包含：文件、位置、问题、建议
- [ ] 严重问题额外包含：代码片段、规范参考
- [ ] 问题编号：每个等级内从 1 开始

#### 总结与建议
- [ ] 优先处理建议：立即处理、本次迭代处理、后续优化
- [ ] 如有亮点：代码亮点章节
- [ ] 如有改进方向：改进方向章节

### 对话摘要格式模板

对话输出必须包含以下内容，与完整报告对应：

```
📋 代码审查完成

**审查时间**：{YYYY-MM-DD HH:mm:ss}
**审查范围**：{路径}
**审查文件数**：{数量}

| 类别 | 严重 | 中等 | 轻微 | 总计 |
|------|------|------|------|------|
| 代码质量 | X | X | X | X |
| 类型安全 | X | X | X | X |
| Vue 规范 | X | X | X | X |
| 样式规范 | X | X | X | X |
| npm 安全 | X | X | X | X |
| **总计** | **X** | **X** | **X** | **X** |

**静态检查结果**：
- ESLint：{通过/发现 X 个问题}
- TypeScript：{通过/发现 X 个错误}

🔴 严重问题：
1. [{文件}:{行号}] {问题描述}
   建议：{修复建议}

📁 详细报告：reports/frontend-code-review-{时间戳}.md
```

### 常见格式错误

| 错误类型 | 说明 | 修正方式 |
|---------|------|---------|
| 章节遗漏 | 缺少必要章节 | 补充缺失章节 |
| 表格格式错误 | 列对不齐、缺少表头 | 重新生成表格 |
| 问题编号错误 | 编号不连续或重复 | 重新编号 |
| 统计数字不一致 | 摘要与问题列表数量不符 | 重新统计 |
| 对话与文件不一致 | 摘要与报告内容不符 | 同步修正 |
